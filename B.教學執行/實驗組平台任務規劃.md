# 實驗組平台任務規劃（RR 平台）

**設計原則：** 學生只需在 Rein Room 介面調整標記 `🔧` 的滑桿數值，按下啟動觀察結果。  
不要求理解數學推導，重點是觀察參數改變後平台的視覺化反應。

平台網址：**reinforcelab.vercel.app**

分為兩堂：
- **Day 1**（4/21）：MAB → Maze2D + Q-table 熱力圖
- **Day 2**（4/28）：CartPole → heli（延伸體驗）→ 圖表互動討論

---

## Day 1

---

### Activity 1 — Multi-Armed Bandit：探索 vs. 利用（30 min）

**平台操作：**
1. 前往 reinforcelab.vercel.app
2. 選擇任務：**Multi-Armed Bandit (MAB)**
3. 切換到 📊 儀錶頁，找到 ε（epsilon）滑桿

**概念說明（螢幕上投影）：**

```
Imagine a row of slot machines.
Each machine has a hidden probability of giving you a reward.

With HIGH ε → the agent tries random machines often  (more exploration)
With LOW  ε → the agent mostly picks the best machine it knows  (more exploitation)
```

---

**Round 1**

| 🔧 Parameter | Value |
|---|---|
| ε (epsilon) | **0.9** |

→ 按 **Start**，觀察約 1 分鐘後按 Pause。

**觀察問題：**
- What does the reward curve look like? Is it noisy or smooth?
- Is the agent converging toward the best machine?

---

**Round 2**

| 🔧 Parameter | Value |
|---|---|
| ε (epsilon) | **0.1** |

→ 按 **Reset**，然後 **Start**，再觀察約 1 分鐘。

**觀察問題：**
- Compare to Round 1: which reward curve is smoother? Which is higher?
- If the agent only exploits (ε ≈ 0), what risk does it face?

---

**課堂討論：**
> "Which ε gave a better average reward in the end?  
> Why might an agent that *never* explores get stuck with a bad strategy?"

---

### Activity 2 — Maze2D：Q-table 熱力圖（50 min）

> ⚠️ Maze1D 由教師示範（演示 γ 的效果），學生不需自己操作。

**平台操作：**
1. 選擇任務：**Maze 2D**
2. 使用預設參數先啟動，觀察 agent 早期行為
3. 按 Pause，切換到 🔬 分析頁，觀察 Q-table 熱力圖

---

**Part A：觀察學習過程**

| 🔧 Parameter | Suggested Value |
|---|---|
| α (alpha)   | **0.5** |
| γ (gamma)   | **0.95** |
| ε (epsilon) | **0.2** |

→ 按 **Start**，等待約 **500–1000 steps**，再 Pause。

**觀察問題：**
- In the early episodes, how does the agent move? (random? systematic?)
- After 1000 steps, does the agent find a shorter path than at the start?

---

**Part B：解讀 Q-table 熱力圖**

切換到 🔬 分析頁。

**觀察問題：**
1. Which area of the maze has the **strongest, most consistent colors**?  
   → What does strong color mean about the agent's confidence?

2. Can you **trace a path from Start to Goal** just by following the arrow colors?  
   → Try it — does it lead to the goal?

3. Are there any cells with **weak or mixed colors**?  
   → Why might those cells be uncertain?

---

**Part C：自由調整（選做）**

| 🔧 Try changing | Suggested test values |
|---|---|
| ε (epsilon) | 0.05 (less explore) vs. 0.5 (more explore) |
| α (alpha)   | 0.1 (slow update) vs. 0.9 (fast update) |

→ 觀察熱力圖的收斂速度是否有變化。

---

**課堂討論：**
> "Look at the cells near the goal versus far from the goal.  
> Which area has clearer arrows? Why does the learning 'spread' from the goal outward?"

## Day 2

---

### Activity 3 — CartPole：連續狀態的學習曲線（50 min）

**平台操作：**
1. 選擇任務：**CartPole**
2. 切換到 📊 儀錶頁
3. 使用預設參數啟動，先觀察早期行為

**概念說明：**

```
CartPole is different from the maze:
- The state has 4 continuous values (position, speed, angle, angular velocity)
- Rein Room automatically "bins" these into discrete states — you don't need to code it.
- Your job: watch the learning curve and observe when improvement begins.
```

---

**Round 1：觀察基礎學習**

| 🔧 Parameter | Value |
|---|---|
| α (alpha)   | **0.5** |
| γ (gamma)   | **0.99** |
| ε (epsilon) | **0.3** |

→ 按 **Start**，持續觀察至少 **50 episodes**。

**觀察問題：**
1. How long does each early episode last? (steps before pole falls)
2. Around which episode do you first notice the agent starting to improve?
3. Describe the shape of the reward curve in **one sentence**.

---

**Round 2：對比高低 ε**

先 Reset，然後：

| 🔧 Change | Value |
|---|---|
| ε (epsilon) | **0.05** (almost no exploration) |

→ 重新啟動，觀察 50 episodes，比較與 Round 1 的差異。

**觀察問題：**
- Did the agent improve faster or slower with low ε?
- Did the reward curve ever suddenly drop? What might cause a sudden drop?

---

**課堂討論：**
> "CartPole is much harder than the maze — the state space is huge.  
> Does the agent eventually solve it? What does 'solved' look like on the reward curve?"

---

### Activity 4 — heli：延伸自由體驗（40 min）

> ⚠️ 此段為延伸體驗，**不列入後測評量**，學生可自由探索。

**平台操作：**
1. 選擇任務：**heli**（直升機飛過間隙）
2. 任意嘗試參數組合，觀察 agent 學習

| 🔧 Parameter | Try anything! |
|---|---|
| ε (epsilon) | high → more random, low → more focused |
| α (alpha)   | affects how fast the Q-table updates |

**開放式探索問題（不需作答，口頭討論即可）：**
- Does the agent ever learn to fly through the gaps?
- What's a good starting ε for this task — high or low? Why?
- Is this easier or harder than CartPole? Why?

### Activity 5 — 圖表互動討論（30 min）

> 教師投影以下情境圖，全班口頭討論（此段不蒐集書面答案）。

---

**討論題 1：先平後升的 reward 曲線**

```
Reward ↑
        |               ／‾‾‾‾‾‾
        |              ／
        |_____________/
        +─────────────────────── Episode
            0        100       200
```

> "What's happening in the first 100 episodes?  
> Why does the reward start rising around episode 100?  
> Does this mean the agent suddenly 'got smart'?"

---

**討論題 2：兩條學習曲線 A vs B**

```
Reward ↑
        |   /\/\/\/\/\  ← Curve A: rises fast, very noisy
        |  /\/\/\/\/\/
        | /‾‾‾‾‾‾‾‾‾‾‾  ← Curve B: rises slowly, smooth
        +──────────────────────── Episode
```

> "Which curve likely has a **higher learning rate α**?  
> Which is more stable? Which would you trust more for deployment?  
> Is there a trade-off between learning speed and stability?"

---

**討論題 3：Maze2D Q-table 熱力圖**

（投影一張 Maze2D 熱力圖，goal 附近顏色深、遠處顏色淺）

> "Which cell does the agent feel most confident about?  
> Why are cells near the goal darker?  
> If you see a cell with no color at all, what does that tell you?"

## 附錄：參數對照快速參考

| 參數 | 全稱 | 影響什麼 | 建議課堂示範值 |
|------|------|---------|--------------|
| α (alpha) | Learning rate | Q 值更新速度 | 0.1 / 0.5 / 0.9 |
| γ (gamma) | Discount factor | 重視遠期 vs 近期報酬 | 0.5 / 0.95 / 0.99 |
| ε (epsilon) | Exploration rate | 探索 vs 利用的比例 | 0.05 / 0.2 / 0.5 / 0.9 |

> α 和 γ 的直覺在 Maze1D 演示時講解，不列入前後測考題。  
> ε 是核心考點（目標 2：探索 vs 利用）。
