### **3.2 教材內容設計與章節安排**

本研究之教材設計採取「**雙平台對齊（Gymnasium × RL Lab）**」的策略： 一方面，透過 Gymnasium 及 Google Colab，讓學生在標準的 Python 環境中直接操作強化學習；另一方面，利用本研究開發的 RL Lab 網頁平台，提供以視覺化與參數操作為核心的任務式學習活動。兩者皆僅需使用瀏覽器即可執行，無須在學生個人電腦上安裝任何程式或套件，符合國中教室中常見的設備條件。

整體教材安排可分為三個階段：

**Gymnasium 入門任務與概念教學**：建立學生對「環境—代理人互動迴圈」的抽象認知，理解 reset()、step() 等關鍵介面。

**RL Lab 任務式學習模組**：以滑桿與圖表為主軸，讓學生在低程式門檻下操作多種任務，體驗強化學習的試錯與策略收斂。

**圖表分析與概念理解練習題設計**：透過圖表判讀、Q-table 視覺化與跨平台對照題，促進學生將觀察轉化為語言化的概念理解。

以下分節詳述各階段之教材內容與教學活動。



### **3.2.1 Gymnasium 入門任務與概念教學**

在教材的第一階段，學生透過 Google Colab 搭配 Gymnasium，體驗標準化的強化學習環境介面。此階段的目標不是「訓練出最佳策略」，而是讓學生能夠清楚回答：

什麼是「狀態（state）」與「動作（action）」？

為什麼每一步都有一個「報酬（reward）」？

什麼是「回合（episode）」？為何需要 reset()？

強化學習中「環境（environment）」與「代理人（agent）」如何互動？

**一、教學情境與工具準備**

教師預先建立一份 Google Colab 筆記本，內容包括：

Gymnasium 套件安裝程式碼（通常一行指令即可，由教師負責維護）。

一個或兩個簡單環境的範例，如：

CartPole-v1（平衡桿任務）

FrozenLake-v1（冰湖格子任務）

基本互動流程程式碼：env.reset()、env.step(action)、env.render()。

簡易統計與繪圖範例，例如紀錄每回合累積報酬並畫出折線圖。

在實際教學時，教師只需將 Colab 連結分享給學生，學生在電腦教室中打開瀏覽器即可投入實作，避免各自安裝 Python、處理相依套件與版本問題。

（圖 3-10 Gymnasium 入門教學 Colab 範例畫面：左側為程式碼區塊，依序呈現安裝、環境建立、互動與繪圖；右側為執行結果，包括簡易的文本輸出與回合累積報酬折線圖）

**圖 3-10 Gymnasium 入門教學之 Colab 範例畫面**

**二、從程式碼認識「環境—代理人互動迴圈」**

教師首先帶領學生閱讀並執行以下程式架構（以偽程式碼概念呈現）：

import gymnasium as gym



# 建立環境

env = gym.make("CartPole-v1")



# 回合迴圈

for episode in range(10):

obs, info = env.reset()     # 重設環境，取得初始狀態

done = False

total_reward = 0



while not done:

action = env.action_space.sample()   # 隨機選擇動作

obs, reward, terminated, truncated, info = env.step(action)

total_reward += reward



if terminated or truncated:

done = True



print("Episode:", episode, "Total Reward:", total_reward)



在此過程中，教師重點說明：

env.reset() 代表「開始新的一局」，環境回到初始狀態，並回傳一個狀態觀測值 obs。

在 while 迴圈中，每一次 env.step(action) 皆代表代理人做出一個動作，環境則回傳：

新狀態 obs

即時報酬 reward

回合是否結束 terminated 或 truncated

回合結束（例如桿子倒下、角色掉入洞中）後，本回合作為一個完整的「學習經驗單元」。

透過實際執行與修改此段程式，學生能從操作經驗中建構「環境—代理人互動迴圈」的心智模型。

**三、加入簡易統計與圖表觀察**

在理解互動流程之後，教材加入圖表呈現。例如，透過在每一回合結束時，紀錄 total_reward 並繪製折線圖：

橫軸：回合編號

縱軸：每回合累積報酬

若動作完全隨機，學生通常會觀察到報酬在某個範圍內上下震盪，並不會明顯成長。教師可藉此提出問題：「如果我們改用某種 ‘學習中的策略’ 來取代亂數，圖會變成什麼樣子？」為後續 Q-learning 與 RL Lab 的操作埋下伏筆。

（圖 3-11 隨機策略下之回合累積報酬折線圖：顯示 10～20 回合的報酬值，多數集中在低分區，波動較大，缺乏明顯上升趨勢）

**圖 3-11 隨機策略在 Gymnasium 中的回合報酬表現**

**四、與 RL Lab 架構之對照**

本階段的最後，教師會引導學生將 Gymnasium 與 RL Lab 進行「概念對齊」：

在 Gymnasium 中：

代理人透過呼叫 env.step(action) 與環境互動。

在 RL Lab 中：

父頁面代表代理人，透過 postMessage 將 action 傳給 iframe 中的環境。

這樣的對照有助於學生理解：實作技術可以是 Python、Colab、網頁或 iframe，但「強化學習的核心架構」在兩個平台上是一致的。



### **3.2.2 RL Lab 任務式學習模組**

教材第二階段轉向本研究的 RL Lab 平台。此階段的主軸是：「在幾乎不寫程式的情況下，讓學生感受到**調參—觀察—修正**的強化學習過程」，並藉由多種任務逐步引出不同的 RL 概念。

**一、任務階梯與對應概念**

RL Lab 的任務設計採階梯式安排，並與 Gymnasium 任務形成互相呼應。典型安排如下：

**多拉霸機（Multi-Armed Bandit）任務**

對應概念：探索與利用（exploration vs. exploitation）。

學生任務：在未知回報機率的前提下，決定要持續拉哪一台拉霸機，以期在有限資源中獲得最大總報酬。

教學重點：解釋為何「一直拉目前看起來最好的機台」可能導致錯失其他高報酬選項，進而自然引出探索的必要性。

**一維迷宮（Maze1D）任務**

對應概念：回合、折扣因子 γ、最短路徑。

學生任務：讓代理人在一條線上的格子中移動，從起點走到終點，途中可能有懲罰區或陷阱。

教學重點：

若 γ 接近 0，代理人只重視立即報酬，可能會選擇短視的行為。

若 γ 較大，代理人會傾向選擇有利於長期獎勵的行動。

**二維迷宮（Maze2D）任務**

對應概念：高維狀態空間、策略圖像化。

學生任務：在格狀迷宮中尋找通往終點的路徑，避開障礙或懲罰區。

教學重點：

透過 Q-table 熱力圖，把「每個位置下最佳動作」視覺化，讓學生認知策略不再只是抽象數字，而是「每一格要往哪個方向走」。

**動態反應任務（例如 Dino、射擊遊戲）**

對應概念：時間敏感、學習難度與樣本數。

學生任務：讓代理人學會跳過障礙、閃避子彈等。

教學重點：

比較靜態迷宮與動態任務的學習曲線，說明「任務越複雜、需要的試錯次數越多」。

**二、學生操作流程與課堂活動設計**

為讓學生在不同任務中有一致的學習體驗，本研究將每個 RL Lab 任務的操作流程設計為固定的幾個步驟：

**任務啟動** 學生在「指南」分頁中點選任務按鈕（如「Maze2D」），系統自動載入對應的 iframe，並在畫面左側顯示任務場景。

**超參數設定** 學生切換至「設定」分頁，調整以下參數：

學習率 α：決定每一步更新 Q 值的幅度。

折扣因子 γ：決定是否重視長期報酬。

探索率 ε：控制代理人選擇隨機動作的比例。

樂觀值 ψ：調整對未來報酬的樂觀或悲觀程度。

延遲時間：影響訓練步伐的視覺速度。

**啟動訓練並觀察行為與圖表** 學生按下啟動按鈕後，代理人開始在任務環境中行動。此時，學生可同步觀察：

任務畫面中代理人的路徑與行為是否有系統性（例如是否逐漸避免陷阱）。

右側即時圖表中的秒級報酬、回合累積報酬與步數變化。

**切換至 Q-table 分頁分析策略** 當訓練進行到一定程度後，學生切換至「分析」或「Q-table」分頁：

查看「動作偏好熱力圖」，理解在不同狀態下代理人的決策傾向。

使用切片工具觀察特定狀態下各動作 Q 值的相對大小。

**調整參數再試一次** 教師引導學生根據圖表觀察提出假設，例如：「如果把 ε 調低，代理人是不是會更 ‘專心’ 在既有策略上？」 學生再回到「設定」分頁調整參數，重新啟動訓練，比較前後差異。

（圖 3-13 RL Lab 操作流程示意圖：以流程圖形式呈現「選擇任務 → 設定參數 → 啟動訓練 → 觀察圖表 → 分析 Q-table → 調整參數再訓練」的循環）

**圖 3-13 RL Lab 任務操作與學習循環流程圖**

**三、Blockly 延伸活動（進階班級）**

對於具備一定程式背景的班級，教師可在後段單元引入 RL Lab 預留的 Blockly 分頁，用以下方式深化學習：

**Rule-based 策略示範** 教師示範如何用積木堆出簡單規則，例如：「在 Maze1D 中，一律往右走；如果撞牆就往左。」學生可直觀看到這種策略在迷宮中的效果，並與 Q-learning 所學得的策略做比較。

**與 Gymnasium 程式碼對照** 教師可將 Blockly 中的決策邏輯翻譯成簡短的 Gymnasium Python 程式片段，讓學生理解「同樣的策略可以有不同的實作形式」，強化抽象思維。

**開放式專題構想** 在課程後期，學生可嘗試設計自己的規則策略，並在 RL Lab 環境中與 Q-learning agent 對戰或比拼表現，增添趣味性與自主性。



### **3.2.3 圖表分析與概念理解練習題設計**

為避免學生只停留在「看畫面覺得好玩」的層次，本研究特別設計一系列圖表分析與概念理解練習題，分別針對：

**時間軸圖表**：如回合累積報酬、步數變化。

**空間圖表**：如 Q-table 熱力圖、動作偏好分布。

**跨平台對照**：針對 Gymnasium 與 RL Lab 的同概念呈現方式做比較。

這些練習題可以編排成紙本學習單或數位題單，搭配教學活動使用。

**一、時間序列圖表判讀題**

此類題目提供來自 Gymnasium 或 RL Lab 的回合報酬、步數圖，要求學生進行解讀與解釋。例題包括：

**例題一：學習率比較** 提供兩條回合報酬曲線（A、B 組），A 組曲線波動較大但上升較快，B 組曲線較平滑但上升緩慢。 問題：

你認為哪一組的學習率 α 較大？

若要使學習過程更穩定但仍希望最終表現不差，你會如何調整 α？

**例題二：探索率的影響** 提供某代理人的步數折線圖，前 50 回合步數上下波動劇烈，之後漸趨穩定而下降。 問題：

圖中哪一段時間代理人「還在嘗試各種走法」？

若將 ε 大幅降低，你預期圖形會出現什麼變化？

**二、Q-table 與策略結構理解題**

針對 Q-table 的視覺化，本研究設計了數種題型，引導學生從「空間分布」的角度理解策略結構：

**例題三：主要動作分布** 提供 Maze2D 的動作偏好熱力圖，每個格子顯示在該狀態下最被看好的動作方向顏色。 問題：

請圈出你認為「接近終點」的區域，說明你如何判斷。

觀察該區域的顏色分布，說明代理人在接近終點時通常會採取哪些行動。

**例題四：策略穩定度與不確定區域** 提供一張「價值差距（gap）」熱力圖，顏色越亮代表最佳與次佳動作 Q 值差距越大。 問題：

如果你是設計者，會如何調整訓練（例如增加訓練回合或調整 ε）來改善這些區域？

**例題五：切片圖解讀** 提供某一固定狀態下，各動作 Q 值的長條圖。 問題：

根據長條圖，哪一個動作目前最被推薦？

如果環境中「獎勵位置」更靠近某一側，你預期這些長條的相對高度會如何改變？

**三、跨平台對照與開放式反思題**

最後，為了讓學生不僅能「看懂單一平台」，還能連結不同實作之間的共通架構，教材設計了若干跨平台對照題與開放式反思題，例如：

**例題六：介面對齊** 題目提供：

Gymnasium 中 obs, reward, done, info = env.step(action) 的程式片段。

RL Lab 中「環境 iframe 回傳 reward_state 訊息、父頁面回傳 action」的流程示意圖。 問題：

請說明這兩種做法在概念上有哪些對應關係。

你覺得哪一種寫法比較容易理解？為什麼？

**例題七：介面選擇與學習感受** 問題：

在 Gymnasium（Colab）中，你是透過程式碼來控制 agent；在 RL Lab 中，你是透過滑桿與按鈕調參數。哪一種方式比較有助於你了解「強化學習正在發生什麼事」？

如果要向完全沒學過程式的同學介紹強化學習，你會選擇先用哪一個平台？請說明理由。

學生在答題過程中，必須將自己在兩個平台上的操作經驗進行比較，並用自然語言整理出「相同之處」與「不同感受」。這種反思活動有助於提升他們對抽象概念的掌握，也為後續第 4 章的教學實驗與質性分析提供豐富素材。
